Welkin layout primitives

Container-query-first, intrinsic layout (docs/06). Each primitive is one class configured by custom-property knobs, shown here in narrow, medium, and wide containers. No viewport breakpoints anywhere on this page.

.stack

Vertical flow with consistent rhythm — the stack owns the space between siblings that the reset removed. Knob: --wel-stack-gap (default --wel-space-4). Rhythm holds at every container width; vary it by nesting stacks, not by per-element margins.

.cluster

Horizontal grouping that wraps: tag lists, button rows, nav items. Knobs: --wel-cluster-gap, --wel-cluster-align, --wel-cluster-justify.

.sidebar-layout

Sidebar + content; wraps to stacked when the content's share would drop below its minimum — no breakpoint. Knobs: --wel-sidebar-width, --wel-sidebar-content-min, --wel-sidebar-gap; data-side="end" puts the sidebar after the content.

.switcher

Row of equals that switches to a column below the threshold. Knobs: --wel-switcher-threshold, --wel-switcher-gap, data-limit caps items per row.

.grid

Auto-fit card grid: as many --wel-grid-min-wide columns as fit. Knobs: --wel-grid-min, --wel-grid-gap.

.center

Measure-capped centred column; a data-breakout child escapes to the full container width via named grid lines. Knobs: --wel-center-max (defaults to --wel-text-measure), --wel-center-gutter.

.cover

Full-height cover with a vertically centred data-principal element; other children keep to the edges. Knobs: --wel-cover-min-height (default 100dvb), --wel-cover-padding.

.frame

Aspect-ratio media frame with object-fit cropping. Knob: --wel-frame-ratio (default 16 / 9).
