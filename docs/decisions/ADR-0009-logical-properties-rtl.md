# ADR-0009 — Logical properties exclusively; RTL = `dir="rtl"` and nothing else

**Status:** Accepted

## Context

Bootstrap ships a separate compiled RTL stylesheet (bootstrap.rtl.css) generated by
RTLCSS — an entire parallel artifact to solve a problem CSS now solves natively. Logical
properties and values are Baseline Widely Available.

## Decision

Welkin uses **logical properties and values exclusively**: `margin-inline`,
`padding-block`, `inset-inline-start`, `border-start-start-radius`, `inline-size`,
`text-align: start`, etc. Physical properties (`left`, `margin-right`, `width` where
`inline-size` is meant…) are banned by lint rule
([04-css-architecture.md](../04-css-architecture.md)).

**Permitted physical exceptions**, each requiring a source comment:

- Shadows and other genuinely physical effects (light does not flip with reading
  direction);
- Media geometry where the axis is intrinsic to the artwork.

RTL support therefore requires from users exactly one thing: `dir="rtl"` on `<html>` (or
any subtree). No RTL build, no `[dir]` selectors in components.

## Consequences

- One stylesheet serves LTR, RTL, and vertical writing modes; mixed-direction subtrees
  (an RTL quote inside an LTR page) just work.
- Zero runtime or build cost; the entire feature is a naming discipline.
- Contributors learn the logical vocabulary; the lint rule teaches it mechanically.

## Alternatives considered

- **Physical properties + generated RTL stylesheet (RTLCSS)** — rejected: parallel
  artifact to build/test/ship, and it can't handle mixed-direction content.
- **Physical properties + `[dir="rtl"]` override rules** — rejected: doubles selector
  surface, guaranteed drift.
